# ADR-002: 国际化（i18n）基建

- 状态：已采纳（基建阶段）
- 日期：2026-06-29

## 背景

平台当前以中文运营文案为主，但 vue-i18n 已在 pure-admin 壳中接入。为后续 en-US 与多区域客户做准备，需要统一键名规范，避免继续硬编码。

## 决策

1. **默认语言**：`zh-CN`（与现有 UI 一致）
2. **新代码**：运营可见文案优先使用 `$t('key')` 或 `t('key')`；存量中文分批迁移
3. **键名规范**：`{域}.{页面}.{元素}`，例如 `seo.overview.todo.title`
4. **第二阶段**：增加 `en-US` locale 文件，不阻塞 MVP 中文交付

## 后果

- 短期增量成本略增（写键 + zh-CN 条目）
- 长期避免大规模字符串替换与遗漏
