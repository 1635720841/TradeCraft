# 本地开发与登录

## 开发账号（执行 seed 后可用）

| 角色 | 邮箱 | 简写 | 密码 |
|------|------|------|------|
| 企业管理员 | `admin@dev.local` | `admin` | `admin123` |
| 企业成员 | `member@dev.local` | `member` | `member123` |
| 超级管理员 | `super@dev.local` | `super` | `super123` |
| 平台管理员 | `ops@dev.local` | `ops` | `ops123` |

> 不是 pure-admin 旧 Mock 的 `admin` + 任意密码，必须走真实后端 API（`POST /api/v1/auth/login`）。

## 首次启动

```bash
pnpm docker:up
cd apps/platform/api
pnpm db:push
pnpm db:seed          # 可重复执行，不会删除已有文章
pnpm dev:api
```

> **注意**：`pnpm db:seed` 默认只补齐开发账号/项目，**不会删 ArticleJob**。  
> 若要强制清空开发环境（含历史文章），才用：`RESET_DEV_DATA=true pnpm db:seed`

另开终端：

```bash
cd apps/platform/web
pnpm dev
```

浏览器打开 `http://localhost:5173`，用上方账号登录。

侧边栏 **平台 → 用量统计** 可查看计费记录；**平台 → Prompt 版本库**（仅 ADMIN）可热更新 Prompt。

首次或升级后请执行：

```bash
cd apps/platform/api
pnpm db:push    # 创建 PromptTemplate 表
pnpm db:seed    # 从 prompts/*.md 导入 7 条模板
```

覆盖已有 Prompt 内容：`FORCE_PROMPT_SEED=true pnpm db:seed`

**切换功能使用的 Prompt 版本**：登录后打开 **平台 → Prompt 运营台**，在对应功能卡片选版本 → 点「应用」（无需改代码、无需重启）。

## 计费用量 API

登录后可用：

```http
GET /api/v1/platform/billing/usage?page=1&limit=20
```

文章任务完成后会自动写入 `CreditUsage`（按篇计费占位）。

## 导出下载

任务详情页「下载 HTML 导出」按钮，或：

```http
GET /api/v1/projects/{projectId}/article-jobs/{jobId}/export/html
```

需 Bearer Token。YMYL 需人工审核时不生成可发布 HTML。

## 登录失败排查

1. 确认 API 已启动：`http://localhost:3000/api/v1/health`
2. 确认已 seed：`cd apps/platform/api && pnpm db:seed`
3. 用户名填上表 **邮箱** 或 **简写**，密码与上表一致（至少 6 位）
4. 浏览器 Network 里登录请求应是 `POST /api/v1/auth/login`（不是 `/login` Mock）
